Issue

  • wip 公開サイトを作る旅 #1228

やったこと

  • related_componentのAPIを作成します。
  • このAPIでデータベースからデータを取得して返す。
  • データ構造：
relatedItem = {
  _id: string;
  name: string;
  meta: MetaModel;
  ogp: OgpModel;
  use_meta_for_ogp: boolean;
  public_time: Date;
  content_updated_at: Date;
  path: string;
  tag_ids: string[];
  _type: string;
  created_at: Date;
  category_id?: string;　　 // 記事だけある
  eyecatch_id?: string;　　// 記事だけある
  intersection?: number;　// 現在ページと同じタグの数
  item_url?: string;　　
  image_url?: string;　// itemのサムネル画像
  tags?: TagData[];　  // itemのタグデータ 
  category?: CategoryData;   // itemのカテゴリデータ。記事だけあります。
};

スクリーンショット

注意

  • APIの確認リンク
    http://homeup.local/cms/api/public_sites/atoms/related_article_component?page_id=${page.id}&site_id={site.id}&sort_by={sort_type}
  • 例えば
    http://homeup.local/cms/api/public_sites/atoms/related_article_component?page_id=60586fc02d931a12265f0f81&site_id=5ddb446703cf1f3edb2feeed&sort_by=public_desc

仕様

  • ページやブログ記事で設定したタグに応じて、同じタグの記事やページを、関連記事として自動でリスト表示させることができます。関連記事パーツが現在ページの関連記事一覧をリスト表示します。
  • リスト順序：リストが２回並べます。2番目の方が優先度が高いです。
    • 1番目：選んだ表示順でリストが並べられます。（例えば：公開日の新しいもの順）
    • 2番目：関連度でリストが並べられます。（関連度が同じタグです。）
